# API-контракт: сайт ↔ ML-сервис

Это **единый договор** между `orevision-app` (наш сайт) и `orevision-ml`
(сервис другой команды). Обе стороны обязаны его соблюдать. Mock и real
ОБЯЗАНЫ возвращать один и тот же JSON — тогда переключение проходит без правок.

> Разделение ответственности (критично):
> **ML отдаёт только «сырые факты о пикселях»** — маску, объекты, площади в
> пикселях, уверенность. **Проценты и класс руды считает САЙТ** (rule-based).
> ML НЕ возвращает `talc_percent` и НЕ решает, оталькованная руда или нет.

## Endpoint

```
POST http://localhost:8001/analyze
Content-Type: multipart/form-data
```

Поля запроса:

| Поле     | Тип        | Обяз. | Описание                                        |
|----------|------------|-------|-------------------------------------------------|
| `image`  | файл       | да    | TIFF / PNG / JPEG изображения шлифа              |
| `params` | JSON-строка| нет   | параметры инференса (порог, тайлинг и т.п.)      |

Дополнительно ожидается `GET /health` → `200 OK` для индикатора в UI.

## Ответ (единый JSON для mock и real)

```jsonc
{
  "model_version": "unet-r34-0.3.1",     // строка версии модели
  "inference_time_ms": 4213,             // время обработки, мс
  "inference_params": { "tile": 1024 },  // с какими параметрами считали
  "image_size": { "width": 10000, "height": 8000 },

  // Тяжёлые данные передаются ФАЙЛАМИ/ПУТЯМИ, не массивами в JSON:
  "mask": "….png",                       // PNG, где ЗНАЧЕНИЕ пикселя = код класса
  "confidence_map": "….png",             // grayscale-PNG, яркость = уверенность

  "class_legend": {                      // расшифровка кодов (см. ниже)
    "0": "фон / нерудная матрица",
    "1": "обычные срастания",
    "2": "тонкие срастания",
    "3": "тальк",
    "4": "артефакт / исключено"
  },

  "objects": [                           // список найденных включений
    { "id": 0, "class": 1, "bbox": [x, y, w, h], "area_px": 12045, "confidence": 0.94 }
  ],

  "warnings": [ "неравномерное освещение в правом верхнем углу" ]
}
```

### Коды классов (class_legend) — ЗАФИКСИРОВАНЫ

| Код | Класс                     | Что это                                                            |
|-----|---------------------------|--------------------------------------------------------------------|
| 0   | фон / нерудная матрица    | силикатная/оксидная основа                                         |
| 1   | обычные срастания         | крупные светлые сульфиды с **небольшим** замещением → рядовая руда |
| 2   | тонкие срастания          | сульфиды со **значительным** замещением серой/тёмной фазой → трудная|
| 3   | тальк                     | тёмная рассеянная фаза в нерудных зонах                            |
| 4   | артефакт / исключено      | царапины, загрязнения, зоны вне анализа                           |

> Труднообогатимость = **степень замещения**, а НЕ размер включения.
> Именно ML решает, класс 1 это или класс 2. Сайт только агрегирует площади.

## Что сайт делает с ответом

1. `ml_client.py` разбирает JSON → объект `MLResponse` (`src/schemas.py`).
2. `metrics.py` читает PNG-маску и считает доли (валидная площадь = всё − класс 4).
3. `classification.py` применяет прозрачные правила → класс руды.
4. UI рисует overlay из маски и показывает метрики/вывод.

## Передача путей: локальный запуск vs Docker

- Если сайт и ML на одной машине/в одном volume — `mask`/`confidence_map`
  это абсолютные пути к файлам, сайт читает их напрямую.
- Если ML в отдельном контейнере без общего диска — договоримся отдавать
  бинарь PNG (base64 или отдельным endpoint `GET /mask/<id>`). Это **v2**,
  фиксируется в HANDOFF.md при интеграции. Для MVP: общий volume/локальная ФС.
