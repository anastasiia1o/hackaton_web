# Переход фронтенда на patch-classification модель (patch-AL)

> **Статус:** ✅ РЕАЛИЗОВАНО (2026-07-04, ветка `stream-a/patch-al-redesign`,
> контракт **v2**). Что сделано: `schemas.PatchGrid`+`MLResponse.patch_grid`,
> валидация `patch_grid` (`contract.py`), блочная маска + patch_grid в
> `mock_ml/generator.py`, единое пространство id разметки (§3), новые
> `src/quantizer.py` и `src/active_query.py`, ImageFolder-экспорт патчей
> (`dataset_export.write_imagefolder` + `dataset_storage.export_active_learning_patch`),
> третий вариант экспорта на странице «Разметка эксперта». 88 тестов зелёных.
> Детали и заметки для потока B — в `docs/coordination/HANDOFF.md` (2026-07-04).
>
> **Статус (исходное предложение):** предложение (v1, черновик для команды).
> **Суть:** ML-модель классифицирует не пиксели/области, а **квадратные патчи**
> train-разрешения. Фронтенд подгоняется под этот формат: ML отдаёт **блочную
> маску** (patch-grid), а active learning превращает выделенную экспертом
> **связную пиксельную область** в набор **кусочно-блочных патчей** для дообучения.
>
> Любое принятое изменение фиксируется в `docs/coordination/HANDOFF.md` и
> бампает версию `API_CONTRACT.md`.

---

## 0. Ключевая идея (почему это дёшево)

Модель — классификатор `патч(окно train-разрешения) → класс + уверенность`.
Панорама режется на сетку патчей, каждый патч получает один класс.

**Главный рычаг совместимости:** сетка патч-классов, растянутая nearest-ом до
полного разрешения, — это **валидная маска классов**, просто кусочно-постоянная
на сетке патчей. Поэтому вся «нижняя» половина сайта, которая читает маску и
считает по ней площади, **не меняется**:

- `src/metrics.py` — считает `class_area_px`/доли по маске → блок-маска для неё
  неотличима от пиксельной;
- `src/classification.py` — правила (`тальк > 10 %` → оталькованная; преобладание
  тонких срастаний → трудная и т.д.) работают на долях патчей как есть;
- рендер overlay в `ui/viewer.py` — рисует ту же маску.

Разделение ответственности из `API_CONTRACT.md` сохраняется:
**ML отдаёт факты о пикселях (теперь — о патчах), сорт руды считает сайт.**

Меняются только: (1) *происхождение* маски (patch-grid вместо пиксельной сети),
(2) *способ разметки* в AL (связная область + один класс вместо пиксельной кисти),
(3) *формат экспорта* для дообучения (ImageFolder патчей вместо imgs+masks).

## 2. Что делает эксперт в AL (уточнённый поток)

Эксперт **по-прежнему выделяет связную пиксельную область** (лассо / залитая
маска), с которой он не согласен, и **ставит ей ровно один класс** (`ordinary` /
`fine` / `talc` / `artifact`). Пиксельную кисть по классам рисовать не нужно —
достаточно контура области и одной метки.

Дальше **фронтенд сам** превращает эту область в кусочно-блочные патчи (см. §5).
Патчи **могут перекрываться** — это осознанно, чтобы из одной области получить
больше обучающих примеров (дешёвая пространственная аугментация).

```
эксперт: обвёл связную область  +  «это talc»
        │  (пиксельный полигон/маска, один class_label)
        ▼
quantizer.quantize_region(mask, L=talc, overlap>0)   ← наша работа
        │  набор перекрывающихся SxS-патчей, все с меткой talc
        ▼
экспорт ImageFolder  →  дообучение классификатора  →  новый /analyze
```

## 3. Классы: единое пространство id

Сейчас коды рассинхронизированы между контрактом и `annotation_config` (в
контракте `talc=3`, в разметке `talc=1` и т.д.). При переходе фиксируем **один**
источник (`src/config.py`), одинаковый для модели, разметки и маски:

| Код | Класс | Роль |
|----:|-------|------|
| 0 | фон / нерудная матрица | не-руда; из `is_background` модели |
| 1 | обычные срастания (`ordinary`) | сорт-признак |
| 2 | тонкие срастания (`fine`) | сорт-признак |
| 3 | тальк (`talc`) | сорт-признак |
| 4 | артефакт / неуверенно (`artifact`) | исключается из валидной площади |

`annotation_classes.json` выставляется в эти же id (файл уже конфигурируем).
Патч с `conf < τ_conf` модель помечает кодом `4`; в разметке класс «uncertain»
не идёт в трейн, а переводит регион в статус `needs_expert_review`.

## 4. Контракт ML: `mask` становится блочной, +`patch_grid`

`mask`, `confidence_map`, `objects[]` остаются валидными (маска блочная). Для
честной передачи квантованной информации добавляется канал:

```jsonc
{
  "model_version": "patchclf-convnext_tiny-1.0",
  "image_size": { "width": 13330, "height": 9489 },
  "inference_params": { "tile": 2272, "stride": 2272, "grid": [6, 9] },

  "mask": "mask.png",                 // = patch_grid.labels, upsample nearest до image_size
  "confidence_map": "conf.png",       // = patch_grid.conf, upsample до image_size

  "patch_grid": {                     // НОВОЕ: сырой квантованный вывод
    "tile": 2272, "stride": 2272,     // окно и шаг в пикселях исходника
    "rows": 6, "cols": 9,
    "origin": [0, 0],                 // левый-верх первого патча в координатах исходника
    "labels": "grid_labels.png",      // rows×cols, пиксель = код класса
    "conf":   "grid_conf.png"         // rows×cols, яркость = уверенность (0..255)
  },

  "objects": [                        // связные компоненты одноклассовых патчей
    { "id": 0, "class": 1, "bbox": [x,y,w,h], "area_px": 41306048, "confidence": 0.91 }
  ],
  "class_legend": { "0":"фон","1":"обычные срастания","2":"тонкие срастания","3":"тальк","4":"артефакт" }
}
```

Валидация (`contract.py`): если `patch_grid` присутствует —
`size(labels) == rows×cols`, коды ∈ `class_legend`, `mask` совпадает с
апскейлом `labels`. `mean_confidence` для `metrics.py` берётся из `grid_conf`
(среднее по валидным, не-`4` патчам).

Сервис `/analyze` — это обёртка инференса (тайлинг → per-patch class/conf →
запись `grid_labels`, `grid_conf`, апскейл в `mask`/`confidence_map`, компоненты
в `objects`). `mock_ml/generator.py` генерит случайный `patch_grid` тем же путём,
чтобы mock и real отдавали идентичный JSON.

## 5. Квантизация области → патчи (`src/quantizer.py`)

**Вход:** пиксельная маска области `M` (связная, от лассо), класс `L`, размер
патча `S` (= одно train-FOV, дефолт 2272), порог покрытия `τ` (≈0.65), шаг
перекрытия `overlap` (>0 — намеренно, для объёма), лимит `N`, `seed=hash(region_id)`.

**Инвариант:** функция **никогда не падает** и **всегда возвращает список**
(возможно пустой) + причину. Патчи режутся из исходной панорамы (не из даунскейла),
метка у всех = `L`.

```python
def quantize_region(M, image, L, S=2272, tau=0.65, overlap=0.5, N=128, seed=0):
    if M.sum() == 0:
        return [], "empty_region"

    # 1) отодвигаемся от границы с соседним классом
    core = erode(M, margin=int(S*(1-tau)/2))
    src  = core if core.sum() > 0 else M          # тонкая область: работаем по M

    patches, stride = [], max(1, int(S*(1-overlap)))   # overlap>0 → перекрытие
    for C in connected_components(src):
        area = C.sum()

        if area >= tau * S*S:                     # --- обычный случай: сетка ---
            centers = grid_over_bbox(C, stride) + jitter(C, seed)   # +рандом-джиттер
            for (y, x) in centers:
                box = clip([x, y, x+S, y+S], image.size)
                inside = overlap_area(C, box) / (S*S)
                if inside >= tau:
                    crop = pad_reflect(image.crop(box), to=S)       # добор у края
                    patches.append(Patch(crop, L, weight=1.0,
                                         inside=inside, upsampled=False, xy=(x, y)))

        elif area >= 0.25 * S*S:                  # --- область меньше патча ---
            s2  = ceil(sqrt(area / tau))          # окно под площадь области
            box = square_around_centroid(C, s2)
            crop = resize(pad_reflect(image.crop(box), s2), to=S)   # апскейл до S
            patches.append(Patch(crop, L, weight=0.5,               # ниже вес
                                 inside=area/(s2*s2), upsampled=True, xy=box[:2]))

        else:                                     # --- слишком мелкая ---
            continue                              # см. фолбэк ниже

    if not patches:                               # тонкая жила / всё отсеялось
        return thin_region_fallback(M, image, L, S, seed)

    return farthest_point_cap(patches, N), "ok"   # разнести, не заводнить
```

### Граничные случаи — «отрабатывает всегда»

| Случай | Что делаем |
|---|---|
| **Область меньше патча** | ужимаем окно `S'` под площадь и **апскейлим до S** (вес 0.5). Согласовано с обучением: модель уже видит down-scale-аугментацию, upsample ей «знаком». |
| **Совсем крошечная** (`< 0.25·S²`) | в трейн не берём (шум/дубли), но не теряем: регион → `needs_expert_review` / метка только для валидации. Возврат — `[]`, причина `too_small`. |
| **Тонкая/вытянутая (жила)** | `thin_region_fallback`: патчи **вдоль медиальной оси**, `τ` смягчаем, но требуем, чтобы **центральное суб-окно** патча лежало внутри `M` (центр не контаминирован соседом); иначе — один центроидный апскейл-патч. |
| **Граница с другим классом** | `erode(M, S·(1-τ)/2)` до семплинга + порог `inside ≥ τ` → патчи из «ядра» области, не со стыка. |
| **Патч вылезает за край изображения** | `clip` + `pad_reflect` до `S`; если реального пикселя `< τ` — отбрасываем. |
| **Несвязные куски / дыры** | обрабатываем покомпонентно; дыры учтены растеризацией `M`. |
| **Огромная область** | `farthest_point_cap(N)` — кап на `N` пространственно разнесённых патчей, чтобы регион не забил датасет near-duplicate'ами. |
| **Перекрытия** | `overlap>0` — намеренно; одна область даёт много патчей. Дедуп — только по идентичным `xy`, не по перекрытию. |
| **Детерминизм** | RNG сидируется `region_id` → экспорт воспроизводим и сверяем. |

Каждый патч несёт provenance: `source_image_id, region_id, x, y, inside, upsampled, weight`.

## 6. Хранилище и экспорт

**`dataset_storage.py`** (на регион): `roi.json` (полигон/маска области в
пикселях панорамы), `class_label`, `status`, `revisions/`. Патчи **не** храним
на диске — только полигон+класс; квантуем детерминированно на экспорте
(компактно, воспроизводимо). Исходные изображения не модифицируются (как сейчас).

**`dataset_export.py`** — формат `ImageFolder`, который ест `train.py`:

```
exports/active_learning_patch/<export_id>/
  imgs/ordinary/<img>_<region>_<k>.jpg
  imgs/fine/...
  imgs/talc/...
  manifest.csv     # path,label,weight,source_image,region,x,y,inside,upsampled
  classes.json
```

В экспорт идут только регионы со статусом `accepted_for_training`
(`EXPORTABLE_STATUSES`).

## 7. Active learning: политика отбора (`src/active_query.py`)

Недостающий «active»: из `patch_grid.conf`/энтропии ранжируем панорамы и регионы —
наверх кластеры низкой уверенности и границы классов. Формируем worklist эксперту
(размечает самое информативное первым); патчи с `conf < τ_conf` авто-помечаются
`needs_expert_review`. Поля статусов под это уже есть — не хватало только политики.

Цикл дообучения:

```
/analyze (patch_grid + conf)
   → active_query: worklist по неопределённости
   → эксперт: связная область + один класс
   → quantizer: перекрывающиеся патчи (робастно)
   → export active_learning_patch (ImageFolder + weights)
   → train.py: base + AL-патчи (взвешенный сэмплинг уже поддержан)
   → model_version++ → redeploy /analyze
```

## 8. Что НЕ меняется

`metrics.py`, `classification.py`, геологические правила и разделение
«ML = факты / сайт = сорт», рендер overlay — **сохраняются**, потому что
блок-маска для них неотличима от пиксельной. Это делает переход минимально
инвазивным для логики, которой дорожит команда.

## 9. Открытые решения

1. **Патчи на диск vs on-the-fly.** Рекомендация — хранить только полигон+класс,
   квантовать на экспорте с сидом от `region_id` (компактно, воспроизводимо).
2. **Размер патча `S`.** Масштаб панорам неизвестен; дефолт `S = 2272` (одно
   ч1-FOV). Возможен мультимасштаб (несколько `S` с агрегацией) — отдельным этапом.
3. **`objects[]`** из компонент патчей — грубые (гранулярность патча). Если нужны
   инстансы включений точнее — это уже отдельная детекция, вне patch-AL.
