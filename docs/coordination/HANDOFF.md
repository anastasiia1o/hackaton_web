## 2026-07-02 — A — Зафиксирован SEAM и контракт (v1)

## 2026-07-02 — A — Коды классов и цвета — единый источник

Что это меняет для другого потока: поток B берёт цвета/названия из config,

## 2026-07-02 — A — Добавлен валидатор контракта + гайд для ML

Что решено: любой ответ ML теперь проходит `src/contract.py` в `ml_client.analyze`
(флаг `config.VALIDATE_ML_RESPONSE`, по умолч. вкл). Для ML-команды написан
`docs/ML_INTEGRATION_GUIDE.md` — человекочитаемое описание контракта.
Что это меняет для другого потока (B): при неверном ответе ML появится
ContractError с понятным текстом — можно показать его в UI как ошибку анализа
(например, `try/except` вокруг `run_analysis`, вывести `st.error(str(e))`).

## 2026-07-03 — B — Интерактивный вьюер (zoom/pan + minimap) и безопасная загрузка панорам

Что решено: в `ui/viewer.py` добавлены `load_display_image()` (decode-downscale
через JPEG `draft`+`thumbnail`, снимает лимит PIL) и `interactive_viewer_html()`
— самодостаточный inline HTML/JS-вьюер с zoom/pan, minimap-навигатором и
кнопками, без внешних CDN. `app.py` теперь грузит панораму уменьшенной (≤2600px
по стороне) и показывает через `st.components.v1.html`; тумблер «Интерактивный
вьюер» в сайдбаре (fallback на `st.image`). `Image.MAX_IMAGE_PIXELS=None`.

Проверено на реальных панорамах из набора (до 27025×21227 ≈ 573 Мп): раньше
`app.py` открывал изображение в полном разрешении и падал/съедал память —
теперь безопасно.

Что это меняет для другого потока (A): overlay/PDF/PNG-экспорт теперь строятся
из УМЕНЬШЕННОГО изображения (сторона ≤2600px), а не из оригинала. Метрики и
классификация не затронуты — они по-прежнему считаются потоком A по маске от ML
(её размер не меняю). SEAM (`schemas.py`) не трогал.

Действие от другого потока: если для отчёта нужна привязка к оригинальному
разрешению — скажи, вынесу `DISPLAY_MAX_DIM` в `config.py` (сейчас константа в
`viewer.py`, чтобы не лезть в твой файл). Полностайловая загрузка исходного
разрешения — отдельная задача B (в `BOARD.md` осталась TODO).

## 2026-07-03 — B — Карта уверенности как переключаемый overlay-слой

Что решено: confidence-map теперь не в expander'е, а полноценный тепловой слой
поверх изображения с тумблером и своей прозрачностью (сайдбар: «Слой
уверенности»). Раскраска: низкая уверенность → красный, высокая → зелёный
(`viewer.colorize_confidence` / `add_confidence_layer` / `confidence_legend`).
Слой композитится в тот же overlay, поэтому работает и в интерактивном вьюере,
и в обычном `st.image`.

Что это меняет для другого потока (A): ничего. Использую `result.ml.
confidence_map_path` как есть (grayscale, ярче = увереннее — как отдаёт mock/ML).
`schemas.py` не трогал.

Действие от другого потока: не требуется.
