# HANDOFF — журнал решений и сообщений между потоками

Хронологический лог. Пиши **только в конец**. Одна запись = одно решение/сообщение.
Шаблон:

```
## YYYY-MM-DD — <автор: A или B> — <короткий заголовок>
Что решено: …
Что это меняет для другого потока: …
Действие от другого потока (если нужно): …
```

---

## 2026-07-02 — A — Зафиксирован SEAM и контракт (v1)

Что решено: структуры `MLResponse / Metrics / Classification / AnalysisResult`
в `src/schemas.py` объявлены стабильными. API с ML — `API_CONTRACT.md` v1.
Что это меняет для другого потока: поток B может писать UI, полагаясь на поля
`AnalysisResult`. Ничего пересчитывать в UI не нужно — всё готово в объекте.
Действие от другого потока: рисовать из `result.metrics` и `result.classification`;
не считать проценты в `app.py`.

## 2026-07-02 — A — Коды классов и цвета — единый источник

Что решено: коды классов (0..4) и цвета overlay живут в `src/config.py`
(`CLASS_NAMES`, `CLASS_COLORS`). Совпадают с `class_legend` из ML.
Что это меняет для другого потока: поток B берёт цвета/названия из config,
не хардкодит их в UI. Легенда — `ui/viewer.legend_items()`.

## 2026-07-02 — A — Тяжёлые данные передаются файлами

Что решено: маска и confidence-map — это ПУТИ к PNG, а не массивы в JSON.
Что это меняет для другого потока: UI грузит маску через
`src.pipeline.load_mask(result.ml.mask_path)`; в JSON их не ищи.
